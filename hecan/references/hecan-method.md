# 中西合参解读方法（双模式，SKILL 问一次）

编排一律走 `scripts/hecan_run.py`（一次拿四系原生输出信封；模型永不自行拼装参数逐系跑）。
各系输出按**各自技能的 method 文件**理解；本文件只管合参层。

## 模式一：共振模式（默认推荐）

① **提炼主题**：从用户问题提炼 2–3 个生命主题（参考下方 hooks 表的主题行；双人固定为
   吸引结构 / 摩擦模式 / 长期功课 三主题）。
② **各系发声**：每主题下，各**可用**体系用自己的词汇说 2–3 句，引用各自盘面因子
   （skipped/degraded 体系如实缺席，不硬补）。
③ **共振与分歧**：标出体系间的主题共振与分歧——分歧同样有价值：两面镜子各说各话，
   是值得自省的张力，不是要抹平的矛盾（如实并陈）。
④ **综合功课**：动力 + 功课 + 一条具体到时间/动作的建议。

## 模式二：全景模式

各可用体系按自己的 method 文件**全程完整解读**（natal 整盘、八字五步、紫微六步、宿曜五步；
双人为 synastry / 合八字七步 / 宿曜五步）→ 终章综合：结构同共振模式③④。

## 红线（一票否决；其余承四系 voice-and-ethics 并集，全部继续生效）

1. **反多重宿命**：多系共振加深的是**主题的分量**，永远不是宿命的确定性。
2. **禁一致性投票**：禁「N 个体系都说 X 所以更准/更真/更注定」——体系间**不互为证据，
   只互为镜子**。
3. **可用性如实**：skipped/degraded 体系如实标注缺席与原因，不以其他体系的结论推补。
4. 各系红线（判死刑词表、血光/破财/病灾、打分/指数、克夫克妻、寿夭）在合参中逐条生效。

## 语言

中文为主；natal 原生英文输出由模型中文转述，引用因子名可保留英文（如 Sun/Capricorn）。
